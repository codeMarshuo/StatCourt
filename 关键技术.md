# StatCourt NBA数据统计平台 - 关键技术

## 一、后端技术

### 1.1 核心框架

| 技术 | 版本 | 说明 |
|------|------|------|
| **Spring Boot** | 3.2 | 核心后端框架，简化配置，快速开发 |
| **MyBatis** | 3.0 | ORM框架，实现数据库持久化操作 |
| **Maven** | 3.9 | 项目构建和依赖管理工具 |

### 1.2 数据库技术

| 技术 | 版本 | 说明 |
|------|------|------|
| **MySQL** | 8.0 | 关系型数据库，存储系统数据 |
| **HikariCP** | 默认 | 高性能数据库连接池 |

### 1.3 Web技术

| 技术 | 说明 |
|------|------|
| **Spring MVC** | Web层框架，处理HTTP请求 |
| **RESTful API** | 接口设计风格，前后端分离 |
| **SSE (Server-Sent Events)** | 服务器推送技术，实现流式响应 |
| **OkHttp** | HTTP客户端，调用外部API |

## 九、技术选型理由

### 9.1 后端技术选型

| 技术 | 选型理由 |
|------|----------|
| Spring Boot | 简化配置，快速开发，生态完善 |
| MyBatis | SQL灵活，易于优化，学习成本低 |
| MySQL | 开源免费，性能稳定，社区活跃 |

### 9.2 前端技术选型

| 技术 | 选型理由 |
|------|----------|
| ECharts | 功能丰富，中文文档，易于上手 |
| Three.js | 3D能力强，社区活跃，效果炫酷 |
| Leaflet | 轻量级，插件丰富，易于集成 |

### 9.3 AI技术选型

| 技术 | 选型理由 |
|------|----------|
| 智谱AI | 国产模型，中文能力强，API友好 |
| SSE | 实时推送，兼容性好，实现简单 |
