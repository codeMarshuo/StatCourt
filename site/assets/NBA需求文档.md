# NBA 球员数据统计平台 产品需求文档

## 一、产品概述

### 1.1 产品定位

本平台是 NBA 中国官方旗下的核心球员赛事数据服务平台，聚焦 NBA 全赛季、全赛事阶段（常规赛、季后赛、总决赛、全明星赛等）的球员个人技术统计数据，为用户提供**权威、实时、多维度、可定制**的球员数据查询、筛选、分析、对比服务，是 NBA 官方赛事数据内容的核心承载载体，也是连接 NBA 赛事与球迷的核心数据桥梁。

### 1.2 产品核心价值

1. **官方权威数据**：数据 100% 来源于 NBA 官方赛事统计体系，实时同步、精准校验，是国内最具公信力的 NBA 球员数据平台。
2. **全维度数据覆盖**：完整覆盖球员攻防两端全技术统计维度，包含基础出场数据、核心得分数据、组织防守数据、投篮效率数据，兼顾普通球迷的基础查询需求与专业用户的深度分析需求。
3. **轻量化与专业化兼顾**：极简的查询路径满足普通用户快速找数据的需求，同时提供高阶筛选、多球员对比、数据可视化等专业功能，覆盖全层级用户场景。
4. **赛事内容生态联动**：球员数据与赛事赛程、战报资讯、高光内容深度联动，实现 “看数据 - 看赛事 - 看内容” 的完整用户闭环。

### 1.3 产品目标

- 短期目标：完成核心球员数据展示、多条件筛选排序功能，覆盖全赛季全赛事阶段数据，满足用户基础数据查询核心需求，保障数据的实时性与准确性。
- 中期目标：完善数据可视化、多球员对比、个性化收藏订阅功能，优化全端用户体验，提升用户留存率与日均使用时长。
- 长期目标：打造国内最权威、用户规模最大的 NBA 球员数据服务平台，联动赛事直播、球迷社区、周边内容，构建完整的 NBA 球迷数据服务生态。

## 二、核心功能定义

### 2.1 核心业务功能

1. **多维度球员数据全景展示**：覆盖 NBA 历史全赛季、全赛事阶段的球员技术统计数据，核心字段包含：排名、球员基础信息、所属球队、比赛场次、场均出场时间、场均得分、场均篮板、场均助攻、场均抢断、场均盖帽、投篮命中率、场均三分命中数、三分命中率、罚球命中率，支持单赛季 / 生涯数据的完整展示。
2. **灵活的筛选与排序能力**：支持按赛季、赛事阶段、所属球队、球员位置进行基础筛选，支持按任意数据字段设置数值区间进行高级筛选，支持所有数据字段的升序 / 降序一键排序，快速定位目标球员与数据。
3. **球员详情深度查询**：单球员全生涯、全赛季的详细数据拆解，包含赛季场均数据、单场比赛明细数据、生涯累计数据、历史排名、荣誉里程碑、赛事相关资讯与高光内容。
4. **多球员数据横向对比**：支持 2-5 名球员的同赛季 / 生涯数据横向对比，通过表格 + 可视化图表直观呈现数据差异，支持对比结果的导出与社交分享。

### 2.2 辅助功能

1. **球员收藏与关注**：用户可收藏心仪球员，自定义分组管理，实时接收球员数据更新、赛事表现里程碑提醒。
2. **数据导出与分享**：支持自定义筛选后的球员数据导出为 Excel/PDF 格式，支持单球员数据卡片、多球员对比结果一键生成分享海报，适配主流社交平台分享。
3. **自定义榜单订阅**：支持用户按数据维度自定义榜单，订阅后定期推送球员排名变化、数据突破提醒。
4. **赛事内容联动**：球员数据与对应赛事战报、新闻资讯、高光集锦深度关联，实现数据与内容的无缝跳转。

### 2.3 后台管理功能

1. **赛事数据全生命周期管理**：支持球员赛事数据的录入、批量导入、自动校验、审核、发布、修改全流程管理，留存完整操作日志。
2. **基础信息库管理**：维护赛季、赛事阶段、球队、球员的基础信息，支持增删改查与批量操作，保障基础信息的准确性。
3. **用户与权限管理**：全平台注册用户信息管理、后台管理员角色与精细化权限分配、操作日志审计。
4. **内容运营管理**：平台首页内容配置、核心榜单推荐、资讯与数据联动、广告位管理、活动运营配置。
5. **系统运维管理**：数据备份与恢复、系统参数配置、安全策略设置、异常监控与排查。

## 四、详细页面功能模块规划

### 4.1 前端用户端页面模块

#### 4.1.1 平台首页

平台核心流量入口，承接用户导航、核心内容触达需求，模块拆解如下：

1. **全局顶部导航栏**：平台 LOGO、核心功能入口（首页、球员数据、球队数据、赛事赛程、资讯新闻）、搜索框、登录 / 注册入口、个人中心入口，滚动页面时吸顶展示。
2. **核心 Banner 轮播区**：赛事热点、官方活动、重磅数据里程碑、平台新功能等核心内容轮播展示。
3. **核心数据榜单区**：分模块展示当前赛季得分榜、篮板榜、助攻榜、抢断榜、盖帽榜 TOP10，每个榜单支持一键跳转至完整球员数据列表页。
4. **热门球员推荐区**：展示当前赛季热度最高、用户关注度最高的球员卡片，卡片包含球员头像、姓名、所属球队、场均核心数据，点击跳转球员详情页。
5. **赛事联动区**：展示当日 / 近期赛事赛程、赛果，关联对应赛事表现突出的球员数据入口。
6. **资讯内容区**：展示最新 NBA 赛事资讯、球员动态、数据深度分析文章，关联对应球员详情入口。
7. **全局页脚**：平台版权信息、NBA 官方链接、隐私政策、用户协议、联系方式、合规备案信息。

#### 4.1.2 球员数据列表核心页（对应目标链接页面）

平台核心功能页，承载全量球员数据的展示、筛选、排序核心能力，模块拆解如下：

1. 面包屑与二级导航区

   - 面包屑导航：首页 > 球员数据统计，支持层级快速返回。
   - 二级功能标签：赛季总览、常规赛数据、季后赛数据、总决赛数据、全明星赛数据、生涯榜单，支持一键切换赛事阶段。

   

2. 基础筛选区

   - 赛季选择下拉框：支持选择 1946 年至今所有 NBA 赛季，默认选中当前最新赛季。
   - 球队筛选下拉框：支持全联盟 30 支球队单选 / 多选，筛选对应球队的球员数据。
   - 球员位置筛选标签：控球后卫 (PG)、得分后卫 (SG)、小前锋 (SF)、大前锋 (PF)、中锋 (C)，支持单选 / 多选。
   - 全局搜索框：支持输入球员姓名、拼音、球队名，实时匹配定位目标球员。

   

3. 高级筛选与排序区

   - 主排序下拉框：默认按场均得分降序排列，支持选择任意数据字段作为排序维度，支持升序 / 降序一键切换。
   - 高级筛选按钮：点击展开筛选面板，支持设置任意数据字段的数值区间（如场均得分≥25 分、出场场次≥10 场），支持多条件组合筛选，提供重置 / 确认按钮。
   - 视图切换按钮：支持表格视图 / 卡片视图一键切换，默认表格视图。
   - 数据统计规则说明入口：点击查看当前赛事阶段的数据统计标准、截止时间、字段释义。

   

4. 球员数据表格主展示区（核心模块）

   - 固定吸顶表头：滚动页面时表头始终置顶，每个字段名称支持点击排序，排序字段旁显示方向箭头，高亮标注。
   - 固定列设计：排名、球员信息列始终固定在左侧，横向滚动时不消失，保障核心信息可视。
   - 表格字段排序（从左到右）：排名、球员信息（头像、姓名、球衣号）、所属球队、比赛场次、场均出场时间、场均得分、场均篮板、场均助攻、场均抢断、场均盖帽、投篮命中率、场均三分命中数、三分命中率、罚球命中率。
   - 行交互规则：鼠标 hover 整行高亮，点击行可快速勾选球员加入对比列表，右键呼出快捷操作菜单（查看详情、加入对比、收藏球员、分享数据）。
   - 异常数据处理：出场场次未达赛事统计标准的球员，数据行标注灰色提示，默认折叠至表格底部，支持点击展开查看。

   

5. 批量操作与分页区

   - 批量操作栏：支持勾选多个球员，一键加入对比、批量收藏、导出选中数据。
   - 分页控制器：支持上一页 / 下一页、页码直接跳转、每页展示条数设置（20 条 / 50 条 / 100 条），默认每页展示 50 条。
   - 全量数据导出按钮：支持将当前筛选条件下的全量数据导出为 Excel/PDF 格式，注册用户可用。

   

#### 4.1.3 球员详情页

单球员全维度数据与信息展示页，模块拆解如下：

1. **球员头部信息区**：球员高清头像、姓名（中文 + 英文）、球衣号码、所属球队、场上位置、身高体重、生日、选秀信息、生涯核心荣誉概览。
2. **核心数据概览区**：当前赛季场均核心数据仪表盘、生涯场均核心数据对比，可视化展示球员核心能力。
3. **赛季数据切换区**：支持切换球员生涯所有赛季，支持常规赛 / 季后赛数据一键切换，展示赛季场均完整数据、单场最高数据、赛季联盟排名。
4. **单场比赛数据区**：展示选中赛季的所有单场比赛明细数据，包含对阵双方、出场时间、全维度技术统计、胜负结果，支持按比赛时间、数据维度筛选排序。
5. **生涯里程碑区**：展示球员生涯数据历史突破、联盟历史排名、获得的所有荣誉与奖项。
6. **关联内容区**：球员最新资讯、赛事高光集锦、相关深度分析文章。
7. **快捷操作栏**：固定在页面底部，包含收藏球员、加入对比、分享主页、数据导出按钮。

#### 4.1.4 数据对比分析页

多球员数据横向对比功能页，模块拆解如下：

1. **对比球员管理区**：展示已选中的 2-5 名对比球员卡片，支持移除球员、添加球员、清空对比列表。
2. **对比维度设置区**：支持选择对比赛季、赛事阶段，自定义勾选需要对比的数据字段，提供全选 / 反选按钮，默认选中核心场均数据。
3. **对比数据表格区**：横向展示所有对比球员的选中数据，高亮标注每项数据的最优值，直观呈现数据差异。
4. **可视化对比区**：支持切换雷达图（攻防综合能力对比）、柱状图（单维度数值对比）、趋势图（多赛季数据变化对比），可视化呈现球员差异。
5. **结果操作区**：支持对比结果一键生成分享海报、导出为 PDF/Excel、复制分享链接、保存对比记录。

#### 4.1.5 个人中心页

注册用户专属功能页，模块拆解如下：

1. **个人信息区**：用户头像、昵称、账号等级、会员状态、资料修改入口。
2. **我的收藏区**：展示用户收藏的球员、球队列表，支持分组管理、快速跳转、取消收藏。
3. **我的订阅区**：展示用户订阅的自定义榜单、数据提醒，支持修改订阅规则、取消订阅。
4. **我的历史记录区**：包含对比记录、浏览记录、导出记录，支持再次查看、重新下载、删除记录。
5. **账号设置区**：密码修改、手机号绑定、隐私设置、消息通知设置、账号安全管理。

### 4.2 后台管理端页面模块

#### 4.2.1 数据管理中心（核心模块）

1. **赛季与赛事管理**：赛季信息增删改查、赛事阶段设置、数据统计规则配置、赛季状态管理（未开始 / 进行中 / 已结束）。

2. **球队与球员信息管理**：球队基础信息（LOGO、名称、赛区、阵容）维护、球员基础信息（头像、姓名、基础资料、所属球队、位置、生涯信息）维护，支持批量导入、单条编辑、审核发布。

3. 赛事数据管理

   - 数据录入模块：支持单场比赛球员数据单条录入、批量 Excel 导入，字段与前端完全匹配，支持数据暂存。
   - 数据审核模块：录入数据需经过校验、审核后才可发布至前端，支持异常标注、驳回修改、审核通过发布。
   - 操作日志模块：所有数据修改操作完整留存日志，记录操作人、操作时间、修改前后内容，支持回溯查询。
   - 批量处理模块：支持全量数据刷新、历史数据修正、数据异常批量排查。

   

4. **高阶数据管理**：进阶数据计算规则配置、自动生成、审核发布。

#### 4.2.2 内容运营管理页

1. 首页运营配置：Banner 图管理、推荐榜单配置、热门球员推荐、首页模块排序与显隐控制。
2. 榜单运营管理：自定义榜单创建、推荐配置、规则设置、历史榜单归档。
3. 内容联动管理：资讯内容与球员 / 球队数据的关联配置、高光内容推荐、活动运营配置。
4. 广告位管理：全平台广告位素材上传、排期配置、上下线管理、数据统计。

#### 4.2.3 用户与权限管理页

1. 注册用户管理：用户账号信息查询、账号状态管理（正常 / 封禁）、用户等级与权限配置、用户行为数据查看。
2. 角色权限管理：后台角色创建、精细化菜单 / 数据 / 操作权限分配、管理员账号管理。
3. 日志审计管理：全平台操作日志、登录日志、异常日志的查询、导出、归档。
4. 用户反馈管理：用户反馈与投诉的接收、处理、进度跟进。

#### 4.2.4 系统设置页

1. 系统参数配置：平台基础参数、数据统计规则、缓存配置、安全策略配置。
2. 数据备份与恢复：数据库定时备份设置、手动备份、数据恢复功能。
3. 系统监控：平台运行状态监控、异常告警、性能优化配置。

## 五、用户界面设计规范

### 5.1 视觉设计规范

#### 5.1.1 色彩体系

#### 5.1.2 字体与排版规范

1. 字体家族

   - 中文：优先使用思源黑体、微软雅黑，保障全平台兼容性；
   - 英文 / 数字：优先使用 Roboto、Arial，采用等宽数字字体，保障数据对齐与可读性。

   

2. 字号与字重规范

   表格

   

   

   

   | 文本类型 | 字号 |     字重      |           应用场景           |
   | :------: | :--: | :-----------: | :--------------------------: |
   | 一级标题 | 24px |  700（粗体）  |  页面主标题、核心数据大数值  |
   | 二级标题 | 20px | 600（半粗体） |      模块标题、卡片标题      |
   | 三级标题 | 16px |      600      |     表格表头、列表项标题     |
   | 正文文本 | 14px |  400（常规）  |    表格正文、常规说明文本    |
   | 辅助文本 | 12px |      400      | 备注信息、标注文本、页脚文本 |

   

3. 排版规则

   - 对齐规则：数字数据全部右对齐，文本内容左对齐，表头与对应内容对齐方式一致；
   - 行高规则：标题行高 1.2，正文行高 1.5，辅助文本行高 1.4；
   - 数据格式：百分比数据保留 1 位小数，场均数据保留 1 位小数，球员姓名中文在前、英文在后括号标注。

   

#### 5.1.3 控件与图标规范

1. **图标规范**：采用 2px 描边线性图标，圆角 2px，基础尺寸 24x24px，语义清晰无歧义，hover 状态同步品牌主色变化，保障全平台风格统一。

2. 核心控件规范

   - 按钮：主按钮（品牌红填充 + 白字，圆角 4px）、次按钮（白填充 + 品牌红描边 + 品牌红字，圆角 4px）、文字按钮（无填充无描边 + 品牌红字），禁用状态透明度 50%；
   - 下拉框：高度 36px，圆角 4px，默认边框 #EEEEEE，聚焦边框品牌主色，下拉面板阴影 0 2px 8px rgba (0,0,0,0.1)；
   - 表格：border-collapse 合并边框，行高 48px，表头背景 #F5F7FA，hover 行背景 #F0F2F5，选中行背景 #FFF0F2；
   - 分页器：页码按钮 32x32px，圆角 4px，选中页码品牌主色填充 + 白字。

   

#### 5.1.4 布局与间距规范

- 栅格系统：PC 端采用 12 栅格布局，基础宽度 1200px，最大宽度 1920px，居中对齐，栅格间距 24px，页面左右边距 20px；
- 间距体系：采用 8px 基准间距，所有间距为 8px 的倍数，模块间间距 32px/40px，卡片内间距 16px/24px，元素间间距 8px/12px/16px；
- 布局优先级：核心数据列固定在左侧，次要数据列支持横向滚动，保障用户第一时间获取核心信息。

### 5.2 交互设计规范

1. **点击反馈**：所有可点击元素 hover 状态有明确视觉变化，点击时有轻微按压反馈，禁用状态无交互效果；
2. **排序交互**：点击表头字段，首次点击降序，再次点击升序，第三次点击取消排序，排序字段旁显示方向箭头并高亮；
3. **筛选交互**：筛选条件选中后实时刷新数据，已选条件以标签形式展示，支持单个标签取消、一键重置所有筛选；
4. **状态反馈**：数据加载时展示骨架屏，按钮点击后显示 loading 状态；无数据 / 无筛选结果时展示空状态插画 + 快捷操作按钮；加载失败时展示错误提示 + 重新加载按钮；
5. **滚动交互**：表格横向滚动时锁定排名、球员信息列，纵向滚动时表头与全局导航吸顶，保障操作便捷性。

### 6.1 核心断点设计

表格







| 设备类型  |   断点宽度    |                         核心适配规则                         |
| :-------: | :-----------: | :----------------------------------------------------------: |
| PC 大屏端 |    ≥1440px    |    12 栅格布局，完整展示所有数据列与功能模块，无内容折叠     |
| PC 中屏端 | 1200px-1439px |   12 栅格布局，完整展示核心数据列，次要数据列支持横向滚动    |

|  移动端   |    <768px     | 4 栅格流式布局，非核心功能全部折叠，表格改为卡片式布局，适配竖屏触摸操作 |

### 6.2 多端适配方案

#### 6.2.1 PC 端适配（核心场景）

- 布局：固定居中 1200px 基础容器，大屏下自适应拉伸，最大宽度 1920px，完整展示所有功能与数据；
- 交互：适配鼠标操作，保留 hover 效果、右键菜单、键盘快捷键等 PC 专属交互，支持表格虚拟滚动处理海量数据渲染。

#### 6.2.2 平板端适配

- 布局：8 栅格流式布局，模块由横向排布改为纵向堆叠，左右边距自适应屏幕尺寸；
- 内容：保留核心筛选条件，高级筛选折叠至下拉面板，表格固定排名、球员、得分、篮板、助攻核心列，其余列横向滚动；
- 交互：适配触摸操作，可点击元素最小尺寸 40x40px，取消 hover 交互，改为点击反馈，优化横向滑动体验。

#### 6.2.3 移动端适配（轻量化场景）

- 布局：4 栅格全屏流式布局，左右边距 16px，所有模块纵向堆叠，全局导航折叠至汉堡菜单；
- 内容适配：仅保留赛季、赛事阶段核心筛选标签，其余筛选条件收纳至抽屉面板；表格视图改为卡片式列表，单卡片展示一名球员的核心数据，点击卡片展开完整数据或跳转详情页；非核心功能折叠至更多菜单；
- 交互适配：可点击元素最小尺寸 44x44px，避免误触；分页器改为滚动懒加载，优化上下滑动体验；弹窗、抽屉采用半屏 / 全屏设计，操作按钮放在屏幕底部拇指操作区。

### 6.3 性能适配策略

- 图片适配：采用响应式图片，不同设备加载不同分辨率的素材，移动端优先加载 webp 格式图片，减少加载耗时；
- 数据加载：移动端采用滚动懒加载，PC 端支持虚拟滚动，避免一次性加载全量数据导致的卡顿；
- 资源加载：不同设备按需加载资源，移动端不加载 PC 端专属的高阶可视化组件，减少首屏加载时间；
- 缓存策略：对赛季、球队、球员基础信息进行本地缓存，减少重复请求，提升二次访问速度。

## 七、完整用户流程设计

### 7.1 用户端核心流程

#### 7.1.1 球员数据查询筛选主流程

1. 流程起点：用户进入平台首页，点击【球员数据】导航入口，进入球员数据列表核心页。

2. 基础筛选：用户选择目标赛季（如 2024-2025 赛季）、赛事阶段（如季后赛），系统自动加载对应数据列表，默认按场均得分降序排列。

3. 精准筛选（多路径并行）：

   - 路径 A：通过球队筛选下拉框选择目标球队，系统刷新展示该球队球员数据；
   - 路径 B：通过位置标签选择目标球员位置，系统刷新展示对应位置球员数据；
   - 路径 C：在搜索框输入球员姓名，系统实时匹配展示目标球员数据；
   - 路径 D：打开高级筛选面板，设置数据区间条件，确认后系统刷新符合条件的球员数据。

   

4. 排序调整：用户点击目标数据字段表头，系统按该字段重新排序，展示最新排名结果。

5. 流程终点：用户查看目标球员数据，可选择跳转详情页、加入对比、收藏球员等后续操作。

#### 7.1.2 多球员数据对比流程

1. 流程起点：用户在球员数据列表页，勾选 2-5 名目标球员，点击【加入对比】按钮，进入数据对比分析页。
2. 对比设置：用户调整对比球员列表，选择对比赛季、赛事阶段，勾选需要对比的数据字段，系统实时刷新对比结果。
3. 结果查看：用户查看对比数据表格，系统高亮每项数据最优值；切换可视化图表，查看雷达图、柱状图、趋势图的直观对比。
4. 结果操作：用户可将对比结果生成分享海报、导出为文件、复制链接分享、保存对比记录。
5. 流程终点：用户完成对比操作，可返回列表页或进行其他操作。

#### 7.1.3 数据导出与分享流程

1. 流程起点：注册用户在球员数据列表页完成筛选后，点击【导出数据】按钮。
2. 导出设置：用户选择导出范围（当前页 / 全量筛选结果）、导出格式（Excel/PDF）、导出数据字段。
3. 文件生成：系统根据设置生成对应数据文件，自动下载至用户本地。
4. 分享流程：用户选择单球员数据 / 对比结果，点击【分享】按钮，选择分享渠道，系统生成分享卡片 / 链接，用户完成分享。
5. 流程终点：用户完成导出 / 分享操作。

### 7.2 管理员端核心流程

#### 7.2.1 赛事数据更新与审核流程

1. 流程起点：单场 NBA 赛事结束，数据运维管理员获取官方赛事球员数据。
2. 数据录入：管理员通过批量导入 Excel 文件或单条录入的方式，将赛事球员数据录入系统，保存至暂存库。
3. 数据校验：系统自动校验数据格式、数值合理性，标注异常数据，管理员对异常数据进行修正。
4. 审核提交：管理员提交修正后的数据至审核环节。
5. 审核处理：审核员对数据进行二次核对，数据准确则审核通过，数据有误则驳回至录入人员修改。
6. 发布上线：审核通过后，管理员点击发布，数据同步至前端平台，更新全平台球员数据。
7. 日志留存：系统自动记录全流程操作人、操作时间、修改内容，完整留存操作日志。
8. 流程终点：数据成功发布，前端用户可查看最新赛事球员数据。
